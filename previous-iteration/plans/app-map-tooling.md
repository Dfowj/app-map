# Plan: App Map tooling — dev/test harness + portable engine

## Context

We're standing up `app-map-tool/` (currently empty, not a git repo) to **develop, iterate, and test** the App Map tooling described in `~/Downloads/app-map-handoff.md`. The App Map is a "map-as-projection" of an app's surfaces (screens, sheets, tabs…), their edges/containment/entry-points, states, and dependencies — refreshed *from* code, with three ownership tiers (deterministic / agentic / manual) and a "warn, record, never block" philosophy.

Two things must live in this repo and stay cleanly separated:
1. **The portable tool** — the reusable core you carry to any new project.
2. **A test harness** — a synthetic sample app + its generated map, so we can run the tool end-to-end and refine output quality *inside this repo*.

**Decisions locked in:**
- Engine in **Python (stdlib + `pyyaml`)**, fronted by a single **`app-map` shell wrapper** that does a dependency preflight (checks `python3` + `pyyaml`, bootstraps a local `.venv` on first run, warns clearly instead of crashing) and dispatches to the engine. The future commit hook calls this same wrapper.
- Test target is a **real, coherent SwiftUI sample app** (`shopmini`) — not stub files. It's built to be pointed at by an *actual agent/skill run* in a later session, so its source must genuinely implement the surfaces/navigation the map describes. A **minimal spec doc** is produced now (saved in-project); the app itself is built in a separate session.
- First pass builds the **deterministic core only**; commit hook and agent skill are deferred.
- **Graphing trimmed:** no visual graph diagram. Navigability comes from surface→surface links. We keep only the cheap parts that *strengthen* linking: **incoming-edge inversion** (backlinks per surface) and **broken-link / stale-anchor detection**. Reachability analysis (orphans, dead-ends, unreachable-from-launch) and any diagram are deferred.
- **Plans live in-project:** planning docs are saved under `app-map-tool/plans/` (not Claude's home dir), and CLAUDE.md documents this convention.

**Portability model:** the reusable unit is the tool root (`app-map` wrapper + `appmap/` + `schema`/`templates` + later `skill/` + `hooks/`). To adopt in a new project: **copy the folder**, run `./app-map init` in the target repo to scaffold its `app-map/` (config + schema + empty surfaces dir), author surfaces, then `./app-map render`. Single dep (`pyyaml`) auto-provisioned by the wrapper → "copy folder and run."

## What gets built (deterministic core — handoff §6 "Deterministic" + §7 A/D/E)

1. **Shell wrapper** `app-map` — resolve tool dir; ensure `python3`; ensure `pyyaml` (create `.venv` + install on first run, else print a one-line install hint); `exec python -m appmap "$@"`.
2. **Schema** `appmap/schema/surface.schema.json` — describes surface frontmatter per handoff §5a (id, title, kind enum, code_anchor, contains, edges[id/to/via/…/note], entry_points, states, dependencies, last_verified, needs_review). Used by hand-rolled validation (no `jsonschema` dep).
3. **Model** `model.py` — dataclasses + frontmatter parse/serialize (split `---` fence → YAML head + markdown body; **body preserved verbatim** = tier-3 safety).
4. **Links** `links.py` — derive incoming edges by inverting all outgoing (powers per-surface backlinks); detect `dangling_links` (edge `to:` a nonexistent id). *No reachability/orphan/dead-end analysis this pass.*
5. **Manifest** `manifest.py` — wholesale rebuild: surface index sorted by `id`, `review_queue` projected from per-surface `needs_review`, `dangling_links`, `map_sha`/`generated_at`.
6. **Validate** `validate.py` — frontmatter shape vs schema; flag stale/missing `code_anchor.file`+`symbol`; resolve screenshot paths against config rules and **warn** (never error) on misses. Everything reports; nothing blocks (exit 0).
7. **Render** `render.py` → `app-map/rendered/`: per-surface HTML (frontmatter + outgoing edges as links + **derived incoming backlinks** + body prose) and a searchable index page. Plain stdlib string templating; no diagram.
8. **CLI** `cli.py`: `init` (scaffold into a target), `render` (wholesale — handoff §7A), `validate`. *Incremental render mode deferred.*

### `plans/shopmini-sample.md` — minimal spec for the real sample app

A small but coherent SwiftUI shopping app. Goal: a codebase an agent can read to assert tier-2 fields authentically. It should be *readable and internally consistent*; compiling/running in a simulator is a nice-to-have, not required for map-tool testing.

**Surfaces & navigation**
| id | kind | code_anchor (symbol) | key edges / containment |
|---|---|---|---|
| `tab-bar` | tab-bar | `RootTabView` | contains `[home, search, cart, profile]`; `launch_surface` |
| `home` | tab-root | `HomeView` | push → `product-detail` (tap product) |
| `search` | tab-root | `SearchView` | push → `product-detail` (tap result) |
| `cart` | tab-root | `CartView` | present sheet → `checkout` (tap Checkout, note: enabled only when total > $0 + address on file); push → `product-detail` (tap line item) |
| `profile` | tab-root | `ProfileView` | (orders list; edges optional) |
| `product-detail` | screen | `ProductDetailView` | "Add to Cart" mutates cart (not an edge) |
| `checkout` | sheet | `CheckoutView` | external dep Stripe; success dismisses (OrderConfirmation modal optional/deferred) |

**States:** `cart` → `default`, `empty`. (Others single-state for now.)
**Entry points:** deepLink `shopmini://cart`; pushNotification "abandoned-cart reminder" → `cart`.
**Data deps:** models `Product`, `Cart`, `Order`; `ProductService` with `GET /products`, `GET /cart` (cart paginated for >50 items). External: `Stripe` (checkout only).

## Verification (end-to-end, inside this repo)

1. `git init`; write `plans/app-map-tooling.md` (this plan) + `plans/shopmini-sample.md` + `CLAUDE.md`; first commit of scaffold.
2. From `samples/shopmini`: `../../app-map validate` → wrapper provisions `.venv` on first run, then a clean report (only screenshot-path warnings).
3. `../../app-map render` → regenerates `app-map/manifest.yaml` and `app-map/rendered/`; open `rendered/index.html` and confirm each surface page shows outgoing links **and** derived backlinks, and body prose is preserved.
4. **Fault injection:** temporarily point one edge `to:` a nonexistent id and delete one `code_anchor.file` reference → re-run `validate`/`render` and confirm `dangling_links` and a stale-anchor warning appear, **exit code stays 0**. Revert.
5. Confirm the wrapper's failure path is friendly: simulate missing `pyyaml` → clear warning, not a stack trace.

This gives a visible, iterable output loop reachable via a single `./app-map` command; next sessions layer the hook and the agent skill on top of this deterministic scaffolding.
